CNR Setup

CNR add new DHCP Clients

Clients此頁面主要用於定義CM上線時Interface所屬的==Client Class==與取得的==Policy==。因此，當一新Cable Moden使用時，必須完成下列程序，才能正確獲取DHCPv6 & DHCPv4 Server 派發的IP Address與==DHCP Option==。而在Clients此頁面中一Interface會有兩則規則如下：

  1. ==1,6,==XX:XX:XX:XX:XX:XX：1,6,後面跟的六段XX為Mac Address。此類規則用於定義DHCPv4 Client Class 與 Policy。

  2. ==00:03:00:01:==XX:XX:XX:XX:XX:XX：00:03:00:01:後面跟的六段XX為Mac Address。此類規則用於定義DHCPv6 Client Class 與 Policy。

Client Class:用於指定DHCP Server所派發之Range，換言之Client所取得之IP Address是由Client Class所決定。
Policy:用於指定DHCP於派發過程中所發出的DHCP Option內容與其相關Provisioning 所需指定之資訊。

接下來將使用Arris TG1672 (Intel Puma6)進行介紹：

  1. 取得板端CM(wan0) MAC Address，本範例中之CM(wan0) MAC Address 為==54:65:DE:22:69:52==

  2. 進入CNR中的Clients頁面

  3. 於上方Name欄位中輸入1,6,==54:65:DE:22:69:52==，並選擇所要指定Client Class (此一欄位會影響DHCP Server 派發之IP Address)，由於本範例所新增的Interface為CM故使用 ==ARRIS_CLIENT_CLASS_CM_BILL==(MUST)。接著接著指定Host Name(Optional)與 Domain Name(Optional)。資訊填寫完成後，點擊Add DHCP Client，進行新增。

  4. 新增完成後，於下方列表中找到上列所新增之Client，並點選該Client進入之Name進入編輯頁面。

  5. 點選Policy name下拉式選單進行編輯，選擇欲為板端指定之Policy，本範例選擇ARRIS_POLICY_CM_BILL。==完成後點擊頁面底端Modify Client==。

  6. 以上範例為新增板端CM Interface的DHCPv4 Client所欲對應的Client Class與Policy。

  7. IPv6的設定方式與IPv4的RuleName如下範例為==1,6,==開頭

    ==1,6,==54:65:DE:22:69:52

  8. 而IPv6的RuleName則如底下範例為==00:03:00:01:==開頭

    ==00:03:00:01:==54:65:DE:22:69:52
